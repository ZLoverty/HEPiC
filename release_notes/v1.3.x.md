# v1.3.x

## v1.3.0

### Klipper 状态监控

- **新增 KlipperStatusWidget**：主页右上角新增 Klipper 状态指示控件，实时显示打印机状态。
  - 彩色圆点指示当前状态：绿色（ready）、橙色（startup）、灰色（shutdown）、红色（error）。
  - 状态异常时显示可展开的详情面板，包含完整错误信息。
  - 提供「固件重启」和「Klipper 重启」按钮，可直接从主页恢复异常状态。
  - 状态正常（ready）时控件自动隐藏，仅在异常时弹出，不占用常规布局空间。

### 连接流程改进

- **兼容 Klipper 异常状态连接**：此前连接自检步骤在 Klipper 处于 `shutdown` 或 `error` 状态时会直接拒绝连接；现在改为显示警告后继续完成连接，使「固件重启」按钮可以正常使用。
- **固件重启指令修正**：`restart_firmware` 改用 Moonraker 专用 RPC 方法 `printer.firmware_restart`，替代原先通过 `printer.gcode.script` 发送 `FIRMWARE_RESTART` 的方式，确保在 shutdown 状态下指令能被正确处理。

### 质检模式

- **G235 挤出力范围更新**：调整 G235 挤出力判定区间。
- **材料数据库更新**：修改材料名称与测试条件，更新挤出力质检标准，将 PLA 改为数组结构，新增第二款材料数据。
- **切换标签页不中断质检**：切换至其他标签页时，质检任务保持运行，不再意外停止。

### 其他

- **修复红外线程关闭卡死**：修复关闭应用时红外成像仪线程偶发卡死的问题。
- **新增测试基础设施**：引入 pytest，添加初始测试配置。
- **发布流程修复**：修复 CI 中 pytz 相关报错，自动从 release_notes 中提取版本说明并写入 GitHub Release 页面。

## v1.3.1

### 传感器自动归零

- **动作序列自动归零**：生成动作序列时，加热完成后会自动执行归零流程，无需手动操作。流程为：回抽 10 mm（F600）→ 等待 2 秒 → 软件归零所有可归零传感器 → 以 F100 慢速送回原位，之后进入三秒倒计时并开始记录。
- **质检模式自动归零**：质检 G-code 中同样在加热完成后、开始挤出前插入相同的归零流程。
- **软件归零 G-code 动作**：新增 `ZERO_SENSORS` 软件动作，通过 `M118 ZERO_SENSORS` 从 G-code 触发，主程序收到后自动对所有可归零传感器清零。

### 视觉图像流重构

- **共享缓冲区 + UI 侧轮询**：VideoWorker、IRWorker、ProcessingWorker 的图像显示逻辑由"每帧 emit 信号"改为"Worker 写入共享缓冲区，UI 统一轮询刷新"。轮询 timer（10 fps）由主窗口管理，彻底避免信号队列积压导致的画面卡顿和滞后。`roi_frame_signal` 仍保留，用于向 ProcessingWorker 投递待处理帧。

### 标签页按需显示

- **相机 / 红外标签页默认隐藏**：「视觉」和「红外」标签页在启动时默认不可见，仅当对应硬件初始化成功后才显示，避免无设备时出现空白页面。

### 稳定性与 Bug 修复

- **修复退出时程序卡死**：退出时现在先调用 `video_worker.stop()` 释放相机资源，再退出线程事件循环；`video_thread` 和 `ir_thread` 的 `wait()` 均加入 3 秒超时，超时后强制 `terminate()`，确保程序能正常退出。
- **数据导入自动清空**：移除"清空数据"按钮，改为导入数据时自动清空旧数据。
- **G-code 文件编码修复**：临时 G-code 文件（`tmp.gcode`）改用 UTF-8 编码写入，修复含中文注释时的乱码问题。
