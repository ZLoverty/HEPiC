# v1.3.x

## v1.3.0

### Klipper 状态监控

- **新增 KlipperStatusWidget**：主页右上角新增 Klipper 状态指示控件，实时显示打印机状态。
  - 彩色圆点指示当前状态：绿色（ready）、橙色（startup）、灰色（shutdown）、红色（error）。
  - 状态异常时显示可展开的详情面板，包含完整错误信息。
  - 提供「固件重启」和「Klipper 重启」按钮，可直接从主页恢复异常状态。
  - 状态正常（ready）时控件自动隐藏，仅在异常时弹出，不占用常规布局空间。

### 连接流程改进

- **兼容 Klipper 异常状态连接**：此前连接自检步骤在 Klipper 处于 `shutdown` 或 `error` 状态时会直接拒绝连接；现在改为显示警告后继续完成连接，使「固件重启」按钮可以正常使用。
- **固件重启指令修正**：`restart_firmware` 改用 Moonraker 专用 RPC 方法 `printer.firmware_restart`，替代原先通过 `printer.gcode.script` 发送 `FIRMWARE_RESTART` 的方式，确保在 shutdown 状态下指令能被正确处理。

### 质检模式

- **G235 挤出力范围更新**：调整 G235 挤出力判定区间。
- **材料数据库更新**：修改材料名称与测试条件，更新挤出力质检标准，将 PLA 改为数组结构，新增第二款材料数据。
- **切换标签页不中断质检**：切换至其他标签页时，质检任务保持运行，不再意外停止。

### 其他

- **修复红外线程关闭卡死**：修复关闭应用时红外成像仪线程偶发卡死的问题。
- **新增测试基础设施**：引入 pytest，添加初始测试配置。
- **发布流程修复**：修复 CI 中 pytz 相关报错，自动从 release_notes 中提取版本说明并写入 GitHub Release 页面。
