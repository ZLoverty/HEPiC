# v1.4.x

## v1.4.0

### 性能优化

- **数据采集频率与 UI 刷新频率解耦**：此前每个数据 tick 都会触发一次完整的图表重绘，采样频率高时界面明显卡顿。现在图表以独立的低频 timer 刷新（上限 15 Hz），数据仍按 `data_frequency` 配置的频率采集。

- **修复图表控件全量 deque 转换**：图表更新时原先调用 `list(full_deque)[-n:]`，会将最多 `final_data_maxlen` 条记录全部复制为列表再截取尾部，随录制时长增长越来越慢。改用 `islice(reversed(deque), n)`，只遍历末尾 n 条，耗时与显示窗口大小成正比，与总记录量无关。

- **用后台线程替代 QTimer 进行数据采集**：`QTimer` 运行在 Qt 主线程时无法可靠维持高采样率——UI 绘制等事件会阻塞事件循环，导致 timer 回调在约 15 ms 内集中爆发。现改用 `_DataCollectorThread`，以 `time.perf_counter()` 精确控制采集间隔，与 Qt 事件循环完全独立。

- **启动时将 Windows 定时器精度设为 1 ms**：通过 `winmm.dll` 调用 `timeBeginPeriod(1)` / `timeEndPeriod(1)`，提高 `QTimer` 和 `time.sleep()` 的触发精度。

### 数据精度

- **时间戳改用墙上时钟**：`time_s` 此前以 `tick 计数 × time_delay` 累加，timer 延迟触发时会产生漂移。现改为 `time.monotonic() - session_start`，无论实际触发频率如何，时间戳均准确反映真实经过时间。

- **图表显示改为固定时间窗口**：图表不再显示固定点数，而是显示可配置的时间窗口（默认 60 秒，通过 `config.json` 中的 `plot_time_window_s` 设置）。采样频率变化时显示范围保持不变。

### 数据记录

- **移除手动写缓冲，依赖操作系统 I/O 缓冲**：此前逻辑将 100 行数据积攒在 `data_tmp` 后再批量写入 CSV，此额外缓冲层是多余的——Python 文件对象已有 8 KB 用户态缓冲，操作系统会自动批量落盘。现改为通过 `csv.writer` 逐行写入，同时移除了 `data_tmp`、批量 `pd.DataFrame` 构造以及热路径对 `pandas` 的依赖。

- **录制开始时打开文件，停止时关闭文件**：避免频繁打开文件句柄，停止录制或关闭应用时确保数据完整落盘。

- **CSV 写入加锁保护**：`_csv_lock` 防止数据采集线程在主线程关闭文件的同时继续写入，避免数据损坏。

### 测试模式

- **`test_mode` 跳过 Moonraker 和 Klipper 检查**：以 `-t` 启动时，连接自检在确认 10001 端口可达后直接放行，跳过第 3、4 步（Moonraker API 检查和 Klipper 状态查询）。`KlipperWorker` 同样跳过 WebSocket 连接，进入空转循环。

### UI

- **录制期间禁用主页鼠标交互**：开始录制时视觉控件的鼠标 ROI 编辑被禁用，停止后恢复，防止实验过程中误操作修改 ROI。

- **数据文件始终保存到桌面**。

## v1.4.1

### 配置文件管理

- **安装版配置文件迁移至 `~/.HEPiC/`**：以 PyInstaller 或 Nuitka 打包运行时，`config.json` 现存放于用户主目录下的 `.HEPiC/` 文件夹，用户无需管理员权限即可修改配置。首次启动时自动将内置默认配置复制到该路径；开发模式（源码直接运行）行为不变，仍使用源码树中的文件。

### 数据处理

- **裁剪参数从"点数"改为"时间窗口（秒）"**：数据处理页的裁剪范围设置由点数改为秒数，语义更直观，且对不同帧率的数据集保持一致的截取效果。

### UI 交互

- **挤出按钮改为长按触发**：主页挤出按钮从单击触发改为按住持续挤出、松手停止，与实际操作习惯更贴合，避免误触。

- **质检页新增历史记录面板**：原先显示静态流程说明的左侧面板替换为动态质检历史列表，每次质检结束后自动追加一条记录（时间戳 + PI 代码 + 均值 ± 标准差），最新记录排在顶部，方便连续多次质检时对比结果。

## v1.4.2

### 界面优化

- **界面标签改为中文**：将主界面各功能区的标签文字统一改为中文，提升可读性。
- **状态栏重构**：重新设计设备状态栏，改用配置文件中的 `label` 键作为状态显示文本，并更准确地反映各设备的实时状态；修复状态控件宽度不固定导致布局抖动的问题。
- **质检页重要数值突出显示**：对质检结果页的关键数值（均值、标准差）进行视觉强调，便于快速读取。
- **Klipper 状态控件移至左下角**：将 Klipper 状态指示控件从右上角移至左下角，位置更合理，不遮挡实验数据。
- **移除多余重启按钮**：移除"草莓"样式的固件重启快捷按钮，现在仅在打印机处于非就绪状态（如 shutdown / error）时才在状态控件中提供重启入口。
- **主页新增进度条**：主页实验序列执行期间显示进度条，并附带预计剩余时间（ETA）；修复进度条不能到达 100% 的问题。
- **未连接设备标签自动隐藏**：摄像头或红外成像仪未成功初始化时，对应的数据标签自动隐藏，避免界面显示无效字段。
- **G-code 控件新增编辑开关**：G-code 显示控件新增编辑模式切换，防止实验过程中误编辑已加载的序列。

### 依赖瘦身

- **移除 matplotlib、skan 依赖**：从项目依赖中移除 matplotlib 和 skan，减小安装包体积。
- **解除对 myimagelib 的依赖**：将项目从 myimagelib 库中解耦，消除外部私有依赖。
- **限制 pandas 版本 < 3.0**：限制 pandas 依赖版本上限，避免 pandas 3.0 引入的 pytz 兼容性报错。
- **打包时明确排除 llvmlite / numba**：在 PyInstaller 打包配置中显式排除 llvmlite 和 numba，防止意外打包这些大型依赖。
