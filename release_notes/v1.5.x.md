# v1.5.x

## v1.5.0

### 材料数据库解耦

- **材料数据库独立仓库化**：材料属性（PLA/PETG/ASA 各牌号的力值范围、温度、速度等）从 HEPiC 代码仓库拆分到独立仓库 [hepic_database](https://github.com/ZLoverty/hepic_database)，更新材料参数不再需要发布新版 HEPiC。
- **启动时自动同步**：`start_app()` 启动时调用 `sync_materials()`，对比本地缓存 `manifest.json` 的 `version` 与 hepic_database 最新 Release 的 tag；版本不一致时下载 `materials.zip`，校验每个 family 文件的 sha256 后原子替换本地缓存目录。网络异常或校验失败时静默回退，不阻塞/中断应用启动（超时上限 5 秒）。
- **本地缓存位置按平台区分**：Windows `%LOCALAPPDATA%\HEPiC\materials`，macOS `~/Library/Application Support/HEPiC/materials`，Linux `~/.local/share/hepic/materials`，可用 `HEPIC_MATERIALS_DIR` 环境变量覆盖。首次运行且缓存为空时，以包内置快照（`HEPiC/database/materials/`）作为种子数据，保证离线也能正常使用。
- **`MaterialDatabase` 暴露数据版本号**：新增 `get_version()`，读取 `manifest.json` 的 `version` 字段；状态栏新增「材料库 vX」常驻显示，方便快速排查当前加载的是哪个版本的材料库。
- **保底回退**：若同步从未成功导致缓存目录为空，`MaterialDatabase` 自动回退读取包内置快照，避免出现空数据库。

### 依赖与打包

- **核心依赖与桌面 UI 依赖拆分**：`pyproject.toml` 将 `PyYAML`、`numpy` 列为核心依赖，`pandas`、`pyside6`、`pyqtgraph`、`opencv-python` 等重量级 UI 依赖移入 `[project.optional-dependencies] ui`，嵌入式设备端引入 HEPiC 作为依赖时无需安装桌面 UI 相关的重依赖。
- **材料 yaml/manifest 数据纳入包打包范围**：`package-data` 增加 `database/materials/*.yaml` 与 `*.json`，确保设备端 `pip install` HEPiC 后也能拿到内置材料快照种子数据。

### 嵌入式设备后端（device/）

- **合并「仪表盘」与「手动控制」页面**：ManualControl 页面并入 Dashboard，减少页面切换，相关控制项整合进 MetricCard 组件。
- **质检模式体验改进**：
  - 新增进度条，覆盖加热、归零、挤出三个阶段——挤出阶段按已耗时/预计耗时估算精确进度，其余阶段按状态文案粗略估算。
  - 图表新增 x 轴与坐标标签；挤出力新增最近 30 秒滚动窗口标准差显示。
  - 新增实时温度、速度显示；任务结束后质检结果（数值、图表）冻结，不再被后续空闲状态覆盖。
  - 修复停止按钮无法正常终止质检任务的问题，并补充了对应的速度、温度/速度冻结显示 bug。
- **设置页新增 IP 地址显示**：新增 `system/info` 接口返回设备局域网 IP，便于现场排查连接问题。
- **界面字号调整**：修正部分控件字号过小的问题。
- **补全前端 `lib/` 目录的版本控制跟踪**：`api.js`、`stores.js`、`ws.js` 此前被 `.gitignore` 排除在外，现已正常纳入版本控制；新增 `upload_static.sh` 静态资源上传脚本。
- **分离 device 仓库**：device 代码不再在主仓库维护，转移至 [hepic_device](https://github.com/ZLoverty/hepic_device)。

## v1.5.1

### 材料数据库同步

- **`manifest.json` 精简**：移除冗余的 `generated_at` 字段，仅保留 `version`。

### 依赖版本

- **放宽 numpy / opencv 版本限制**：`numpy` 由固定 `==1.26` 改为 `>=2.0`，`opencv-python` 由固定 `==4.8.1.78` 改为 `>=4.10`，`pandas` 下限提升至 `>=2.2.2`，以便支持 Python > 3.12 环境（此前误以为 Optris SDK 强制要求旧版 numpy/opencv）。待确认与 Optris SDK 的兼容性。

## v1.5.2

### 材料数据库同步

- **新增腾讯云 COS 备用同步通道**：当 GitHub 无法访问时（例如境内网络对 GitHub 访问受限/较慢），`sync_materials()` 自动切换到腾讯云 COS 镜像下载材料库（固定的 `manifest.json` / `materials.zip` / `materials.zip.sha256` 三个对象），下载后同样校验 sha256 通过才原子替换本地缓存；仅在 GitHub 本身不可达时才会触发该回退，而非本地已是最新版本时也去请求 COS。
- **修复缓存目录曾回退到内置快照后无法再次生效同步结果的问题**：`MaterialDatabase.load()` 此前会复用上一次已回退为内置快照的 `materials_dir`，导致 `sync_materials()` 后续同步成功、缓存已更新，再次调用 `load()` 时仍读取的是旧的内置快照数据；现在每次 `load()` 都会重新从配置目录解析，确保读到最新同步结果。

### 质检模式

- **质检历史记录持久化到本地 SQLite**：新增 `qc_history_store`，质检完成后的记录（时间、材料、pi_code、力值均值/标准差）保存到本机 SQLite 数据库（`qc_history.sqlite3`），不再随应用重启丢失；HEPiC 桌面端与 hepic_device 嵌入式后端共用该实现，各自在本机保存独立的历史文件。
- **历史面板改为按日期分组显示**：加粗日期标题分隔不同日期的记录，便于查看跨天的历史数据。
